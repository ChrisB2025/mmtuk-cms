Getting Started

A quick orientation for new editors and contributors.

What this CMS does

The MMTUK CMS manages the content on mmtuk.org. Every change you make — a new article, an updated page section, an uploaded image — is saved to the database and goes live on the site immediately. No deploy step is needed.

The sidebar

The sidebar on the left is your main navigation. It has three tabs at the top:

  • Chat — Create new content by talking to Claude, the AI assistant
  • Content — Browse, search, edit, and delete all existing site content
  • Pages — Edit the text on static pages (Home, Research, etc.) — editors and admins only

The sidebar footer links give you access to the Media Library, Pending Approvals, Activity Log, and Health Check (visibility depends on your role).

Quick navigation — Ctrl K

Press Ctrl+K (or Cmd+K on Mac) from anywhere to open the command palette. Start typing the name of any page or action to jump to it instantly.

Dark mode

Click the sun/moon icon in the top-left corner of the sidebar to switch between light and dark themes. Your preference is saved automatically.

Keyboard shortcuts

Ctrl+K opens the command palette from any page. You can also press Escape to close any open modal or dialog.

Group Leads

Group Leads are tied to a specific local group. Your assigned group determines which events and news items you can approve and publish directly. The six local groups are: Brighton, London, Oxford, Pennines, Scotland, Solent.

As a Group Lead you can create local events and news for any group, but you can only directly publish content for your own group. Content for other groups goes into the approval queue for an editor or admin to review.

Changing your role

Roles are set by an admin via the Django admin panel (/admin/). If you think your role needs changing, contact the site administrator.

Creating Content

All content is created through a chat conversation with Claude. There are three ways to start.

A — Write something new (natural language)
  1. Click + New Chat in the sidebar, or press Ctrl+K and select "New Chat"
  2. Tell Claude what you want to create — e.g. "I want to write an article about the Job Guarantee"
  3. Claude will ask you a few follow-up questions (title, category, author, date)
  4. Review the draft it proposes in the chat — ask for any changes
  5. Say "looks good" or "create it" to confirm
  6. If you can publish directly, it goes live. Otherwise it's submitted for approval
B — Import from a URL (Substack, blogs, etc.)
  1. Start a new chat
  2. Tell Claude to add or import the article — e.g. "Add this briefing: https://example.substack.com/p/article"
  3. Claude scrapes the title, author, date, body, and header image
  4. Review the preview — check title, author, and publication date are correct
  5. Say "yes" or "create it" to confirm
  6. Content is created with full source attribution automatically included
Tip: If you paste a URL without asking Claude to import it, it will be treated as a normal conversation — Claude won't automatically try to create a briefing from it. Use words like "add", "import", or "create briefing from" to trigger the import workflow.
C — Upload a document (PDF or Word, max 20 MB)
  1. Start a new chat
  2. Click the paperclip icon in the chat input and select your file
  3. Claude extracts the text and images, and proposes how to format it on the site
  4. Review the proposed title, content type, and slug — make any corrections
  5. Confirm with "looks good" — Claude creates the content
  6. Images from the document can be embedded; Claude will reference them automatically

What happens after you submit

Editors and admins: Content publishes immediately and is live on the site. You'll see a confirmation message in the chat.

Contributors and Group Leads (outside own group): Content is saved as a draft and sent for approval. You'll see "Your [type] has been saved and is awaiting approval." Watch for the notification on your next login.

Editing existing content

Go to the Content tab, find the item, and click Edit in Chat. A new conversation opens with the current content loaded. Tell Claude what to change — it will show you a diff and ask for confirmation before saving.

For quick frontmatter edits (changing a title, date, or featured flag without touching the body), use the Quick Edit option instead — it opens an inline form.

Deleting content

Find the item in the Content browser and click Delete. You'll be asked to confirm and can optionally set a redirect — visitors to the old URL will be sent to the page you choose (e.g. the articles index). This preserves SEO and avoids broken links. Deletion is permanent — editors and admins only.

Content Types

There are seven content types on the MMTUK site. Use this as a quick reference when Claude asks you which type to create.

Article Most common

In-depth educational pieces, commentary, and research. The bulk of the site's content.

  • Category (required) — see Editorial Guide for which to choose
  • Author — defaults to "MMTUK" if not specified
  • Read time — estimated minutes; defaults to 5
  • Featured — mark to surface the article in homepage and section hero slots
Appears on: /research (Research category) · /education (Core Ideas, Core Insights, But what about...?) · linked from other articles
Briefing Long-form

Longer, more rigorous analysis — often from external MMT economists or reprinted from Substack. Use URL import to create these quickly.

  • Source fields — sourceUrl, sourcePublication, sourceAuthor — filled automatically on URL import
  • Typical read time: 8–15 minutes
Appears on: /research — the three most recent briefings are shown in the Briefings section
News Announcements

Organisational news, press releases, and event announcements at a national level.

  • Category: Announcement · Event · Press Release · Update
Appears on: /about-us → News section
Local Event Group content

Events run by or for a local group — meetups, workshops, screenings, panels.

  • Local group (required): brighton · london · oxford · pennines · scotland · solent
  • Date & location — both required
  • Event type tag — e.g. "Workshop", "Meetup", "Screening"
  • Events auto-archive 7 days after their end date
Appears on: /community and /local-group/[group]
Local News Group content

Updates, announcements, and short news items from a specific local chapter.

  • Local group (required) — same six groups as local events
  • Shorter than national news — 1–3 paragraphs is typical
Appears on: /community and /local-group/[group]
Bio Admin only

Team member and advisor profiles. Only admins can create bios.

  • Includes name, role, photo, and optional LinkedIn/website links
  • Mark advisoryBoard: true to appear in the Advisory Board section
Appears on: /about-us → Steering Group and Advisory Board sections
Ecosystem Entry Directory

Organisations, projects, and allies in the broader MMT ecosystem — think tanks, academic groups, advocacy organisations.

  • Includes website, country, and type classification
  • No public page yet — coming soon

Approvals

Content from contributors goes through an approval step before it appears on the site.

For Contributors — what happens to your drafts

  1. You submit content via chat
  2. You see "Your [type] has been saved and is awaiting approval"
  3. An editor or admin reviews it in the Pending Approvals dashboard
  4. If approved — you'll see "Your [title] has been approved and published!" on your next login
  5. If rejected — you'll see the feedback message explaining what to change. Start a new chat to revise and resubmit
There is no editing of an existing draft. If your draft is rejected, create a new conversation and resubmit the corrected version.

For Editors and Admins — how to approve

  1. Click Pending Approvals in the sidebar footer
  2. The list shows: who created it, content type, when submitted
  3. Click Review to open the detail view
  4. Left panel: metadata (title, type, author, slug, date)
  5. Right panel: full markdown preview — exactly how it will appear on the site
  6. Check: Does it meet our editorial standards? Is the category correct? Is the slug sensible?
  7. Click Approve & Publish — it goes live on the site immediately
  8. Or click Reject — enter feedback; the contributor is notified on their next login

For Group Leads

You can only approve local events and local news from your assigned group. Content from other groups will appear in the Pending list but the Approve button will be disabled.

Giving useful rejection feedback

Good rejection feedback is specific. Instead of "not suitable", try:

  • "Title needs to be more specific — try 'How Bank of England Rate Decisions Don't Control Inflation'"
  • "Category should be 'Commentary' not 'Research' — this is opinion not a research paper"
  • "Missing source attribution — where does the data in paragraph 3 come from?"

Editorial Guide

What MMTUK publishes, and why. Read this before creating content.

Our purpose

MMTUK exists to explain how money and the UK economy actually work, and to connect that understanding to progressive policy. Our content should always:

  • Be grounded in institutional reality — how the Bank of England, HM Treasury, and HMRC actually operate; not how mainstream textbooks say they do
  • Be UK-specific — reference pound sterling, Bank Rate, OBR, ONS, fiscal rules; not generic "the government" without context
  • Connect theory to policy — every piece of theory should land in a concrete policy implication (Job Guarantee, Green New Deal, ZIRP, public investment, etc.)
  • Be accessible — no assumed economics background; jargon explained on first use

Article categories — which one to use

CategoryUse whenTypical length
Core Ideas Explaining a foundational MMT concept for the first time — e.g. "What is a Job Guarantee?" or "Why taxes don't fund spending" 4–6 min read
Core Insights A sharper, more specific insight — e.g. "Why the BoE can't run out of reserves" or "The accounting identity behind the deficit" 3–5 min read
Article Analysis, opinion, or current affairs viewed through an MMT lens — broader than a single insight 5–8 min read
Commentary Topical response to a news event or policy announcement — time-sensitive 3–5 min read
Research Academic-style, evidence-heavy, longer-form — cites sources, uses data, makes a scholarly argument 8–15 min read
But what about...? Directly rebutting a named common objection to MMT — must use rebuttal layout 3–6 min read

What we need more of

  1. "But what about...?" rebuttals — the Education page lists 10 objection topics that each need a linked article. These are high priority.
  2. Local group event write-ups — after every meetup or event, a short local news item keeps the Community page active
  3. Briefings from UK-based economists and writers — import from Substack via URL; source attribution is automatic
  4. Core Insights tied to current UK news — e.g. an insight on Bank Rate after an MPC announcement

What we don't publish

  • Pure party political analysis — MMTUK is cross-party
  • Content that treats government finances like a household budget without immediately correcting it
  • Ideological advocacy that isn't grounded in how the monetary system actually works
  • Speculation presented as fact — if it's contested, say so

Style Guide

How MMTUK content is written — voice, structure, and UK conventions.

Voice and tone

  • Direct and declarative — "The government creates money when it spends" not "Some economists argue that..."
  • Confident in heterodoxy — we challenge mainstream economics; we don't hedge our core claims with "arguably" or "perhaps"
  • Plain language — a motivated sixth-former should be able to follow the argument from beginning to end
  • Precise — accounting identities, operational flows, and institutional detail matter; get them right
  • Not preachy — explain the mechanics; let readers draw their own policy conclusions
  • Not tribal — we engage seriously with counterarguments rather than dismissing them

Structural patterns that work

  • Open with the mainstream claim or myth — then show why it doesn't describe how things actually work
  • Use numbered steps to walk through a monetary operation (e.g. "How government spending creates bank reserves, step by step")
  • Include a short "In a nutshell" summary before a long technical section — not just at the end
  • Anticipate the obvious objection and address it explicitly — don't wait for a comment section
  • Close with the policy implication — what changes if this understanding is correct?

Headings

Use H2 headings only. The site's CSS renders H3 and below the same size as H2, so adding a third heading level creates visual confusion with no benefit. If you need a sub-structure within a section, use bold text or a definition list instead.

Links

  • Link to related MMTUK articles wherever relevant — this builds the internal knowledge structure
  • When quoting external sources, always link to the original
  • Don't link to paywalled content without noting that it's paywalled

UK English conventions

  • Organisation (not organization); labour (the economic factor); Labour (the political party)
  • £ not $ — we write about the UK economy
  • "Chancellor" not "Treasury Secretary"; "Bank Rate" not "Fed Funds Rate"
  • Name UK institutions: BoE, OBR, ONS, HMRC, HM Treasury — not generic "the central bank" or "the government"
  • Dates: 25 February 2026 (not Feb 25 or 25/02/26)

Images

  • 16:9 ratio for article thumbnails and briefing covers
  • Square (1:1) for bio photos
  • Use descriptive alt text — required for accessibility and SEO
  • Avoid stock photos of coin stacks, vault doors, or piggy banks — they reinforce exactly the wrong mental models about money
  • Prefer charts, diagrams, infographics, and event photography

Numbers and statistics

  • Spell out numbers one through nine; use numerals for 10 and above
  • Use £bn and £tn for large figures (£200bn, £2tn) — not "£200 billion"
  • Always cite the source of statistics; link where possible

Page Manager

Edit the text on MMTUK's static pages — Home, Research, Education, Community, and more.

Editors and Admins only. The Pages tab is not visible to contributors or group leads.

How to edit a page

  1. Click the Pages tab in the sidebar
  2. Select the page you want to edit
  3. You'll see a list of sections (Hero, Features, FAQ, etc.)
  4. Click Edit next to the section you want to change
  5. Fill in or update the fields
  6. Click Save & Publish — the change is live on the site immediately

Field types

TypeHow it works
Text field Single line of text — for headlines, labels, button text, short descriptions
Markdown field Rich text with a WYSIWYG editor — supports bold, italic, headings, lists, and links. Use H2 only (see Style Guide)
Text list One item per line in a textarea — used for bullet lists or tag groups
Item list Repeating structured items — e.g. FAQ entries, feature cards, team member cards. Use the + / − buttons to add or remove items

Admin-only fields

Site Config is admin-only and contains sensitive integration fields: Discord invite URL, Stripe payment links (Supporter, Founder, Patron tiers), and the Action Network form ID. These are not visible to editors.

What the Page Manager can't do

  • Add new sections to a page — that requires a code change
  • Change the page layout, design, or component structure
  • Upload images — use the Media Library for that, then paste the image path into the relevant field

Media Library

All images and files used on the site are stored here.

Admins, Editors, and Group Leads only. Contributors cannot upload or manage media.

Uploading an image

  1. Click Media Library in the sidebar footer
  2. Click Upload Image
  3. Select a file — JPG, PNG, WebP, or GIF, max 10 MB
  4. The image is uploaded and a path like /images/my-image.webp is shown
  5. Copy that path and paste it into any image field in a content form or page editor

Image guidelines

PropertyGuidance
Format The CMS accepts JPG, PNG, and WebP. Uploaded images via chat are automatically optimised and converted to WebP for smaller file sizes
File size Under 500 KB for article thumbnails. Hero images up to 1 MB. Compress before uploading if needed
Aspect ratio 16:9 for article thumbnails and briefing covers. Square (1:1) for bio photos. 3:2 for event images
Filename Descriptive, hyphenated lowercase: job-guarantee-paper-cover.webp not IMG_4521.JPG
Alt text Always write descriptive alt text when the form asks for it — required for accessibility and SEO

Deleting images

Images can be deleted from the Media Library, but check first that they are not in use on any page or article. Deleting an image that is referenced in content will cause a broken image on the live site.

Activity Log

A chronological audit trail of every content change on the site.

Editors and Admins only. The Activity Log is not visible to contributors or group leads.

What it shows

  • Who made the change (user name and role)
  • What changed (content type, title, action — create, edit, delete)
  • When the change was made
  • For deletions: whether a redirect was set and where it points

When to use it

  • Checking who last edited a piece of content
  • Verifying that a contributor's draft was approved (and by whom)
  • Reviewing recent site changes after a handover or holiday
  • Investigating unexpected content changes

Health Check

A diagnostic page that verifies the site is working correctly.

Admins only. The Health Check is not visible to other roles.

What it checks

  • Database connection — can the site reach PostgreSQL?
  • Content counts — how many articles, briefings, news items, events, and bios exist
  • Page data files — are all JSON page data files present and valid?
  • Static files — can the site serve CSS, JS, and images?

When to use it

If something seems wrong with the site — pages not loading, content missing, or errors appearing — check the Health Check first. It will flag the specific component that is failing.
